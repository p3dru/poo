# P.O.O. | O.O.P.
## Repositório destinado à explicar POO de forma simples utilizando Python como linguagem de programação 

#### Conceitos rápidos:
- _**Classes:**_ Uma estrutura que define um tipo de dado abstrato com atributos (características) e métodos (ações). Classes podem ser entendidas como abstrações de “coisas do mundo real” como, por exemplo, uma pessoa que tem atributos/características como nome, idade, sexo, cor dos olhos… e também possui métodos/ações como andar, correr, pular, falar…
- _**Objeto:**_ Instância de classe com seus próprios valores de atributo. Exemplo de uma instância da classe Pessoa: Paulo, 20 anos, masculino, olhos castanhos… como uma pessoa, ele pode andar, correr, pular, falar… esses são seus métodos.
- _**Encapsulamento:**_ “Ocultar detalhes de implementação de um objeto, expondo apenas uma interface pública para interação”. Em resumo, esse conceito informa que não é uma boa prática deixar todas as informações expostas para os usuários, para modificar  o acesso de variáveis e métodos, temos as palavras reservadas “Public”, “Protected” e “Private”. [Exemplo de Encapsulamento](https://github.com/p3dru/poo/blob/main/pessoa_encapsulamento.py)
  - **Public:** Onde todos podem acessar (todo o código e códigos que o incorporam).
  - **Protected:** Onde todas as classes que estiverem no mesmo pacote podem acessar a classe que possui o atributo.
  - **Private:** Onde a única classe que tem acesso ao atributo é a própria classe que o define, ou seja, se uma classe Pessoa declara um atributo privado chamado nome, somente a classe Pessoa terá acesso a ele.
- _**Herança:**_ “Permite que uma classe herde atributos/características e métodos/ações de outra classe”. É como herança genética, uma criança herda genes de seus pais, se uma classe é “filha” de outra, faz sentido ela herdar traços (atributos e métodos) da classe “pai”. [Exemplo de herança](https://github.com/p3dru/poo/blob/main/veiculo_heranca.py)
  - _**Herança múltipla:**_ Onde uma classe herda características e comportamentos de mais de uma classe. Tem a vantagem da reutilização de código e flexibilidade, com a desvantagem, de poder aumentar a complexidade do código e gerar ambiguidade. (é importante definir bem escopos para utilizá-la).
- _**Polimorfismo:**_ “Permite que um objeto se comporte de diferentes maneiras, de acordo com o contexto”. É implementado através de herança e sobrecarga de operadores. Exemplo, em uma classe, podemos criar um método genérico que pode se ajustar de acordo com a implementação das classes filhas (sobrecarga de operadores). Citar exemplo em código linkando no readme. Permite que o código seja mais flexível e reutilizável, uma vez que permite que diferentes classes sejam tratadas como se fossem da mesma classe base, desde que implementem os métodos necessários. [Exemplo de polimorfismo](https://github.com/p3dru/poo/blob/main/animal_polimorfismo.py)
- _**Abstração/Interfaces:**_ “Consiste em criar modelos que representam conceitos do mundo real, ignorando detalhes irrelevantes”. É implementada através de classes e interfaces onde as classes servem como modelos que definem a estrutura e o comportamento de objetos e as interfaces definem o contrato que as classes devem implementar. [Exemplo de abstração](https://github.com/p3dru/poo/blob/main/veiculo_abstracao.py)

## Repository designed to explain OOP in a simple way using Python as a programming language
#### Quick concepts:
- _**Classes:**_ A structure that defines an abstract data type with attributes (characteristics) and methods (actions). Classes can be understood as abstractions of “real world things” such as, for example, a person who has attributes/characteristics such as name, age, sex, eye color… and also has methods/actions such as walking, running, jumping, talking…
- _**Object:**_ Class instance with its own attribute values. Example of an instance of the Person class: Paulo, 20 years old, male, brown eyes… like a person, he can walk, run, jump, talk… these are his methods.
- _**Encapsulation:**_ “Hide implementation details of an object, exposing only a public interface for interaction”. In summary, this concept informs that it is not a good practice to leave all information exposed to users, to modify the access of variables and methods, we have the reserved words “Public”, “Protected” and “Private”.
  - **Public:** Where everyone can access (all code and codes that incorporate it). [Encapsulation Example](https://github.com/p3dru/poo/blob/main/pessoa_encapsulamento.py)
  - **Protected:** Where all classes that are in the same package can access the class that has the attribute.
  - **Private:** Where the only class that has access to the attribute is the class that defines it, that is, if a Pessoa class declares a private attribute called name, only the Pessoa class will have access to it.
- _**Inheritance:**_ “Allows a class to inherit attributes/characteristics and methods/actions from another class”. It's like genetic inheritance, a child inherits genes from its parents, if a class is a “daughter” of another, it makes sense for it to inherit traits (attributes and methods) from the “parent” class. [Inheritance example](https://github.com/p3dru/poo/blob/main/veiculo_heranca.py)
- _**Multiple inheritance:**_ Where a class inherits characteristics and behaviors from more than one class. It has the advantage of code reuse and flexibility, with the disadvantage of increasing code complexity and generating ambiguity. (it is important to define the scope to use it well).
- _**Polymorphism:**_ “Allows an object to behave in different ways, according to the context”. It is implemented through inheritance and operator overloading. Example, in a class, we can create a generic method that can adjust according to the implementation of the child classes (operator overloading). Cite an example in code by linking in the readme. It allows code to be more flexible and reusable, as it allows different classes to be treated as if they were the same base class, as long as they implement the necessary methods. [Example of polymorphism](https://github.com/p3dru/poo/blob/main/animal_polimorfismo.py)
- _**Abstraction/Interfaces:**_ “Consists of creating models that represent real-world concepts, ignoring irrelevant details”. It is implemented through classes and interfaces where classes serve as models that define the structure and behavior of objects and interfaces define the contract that classes must implement. [Example of abstraction](https://github.com/p3dru/poo/blob/main/veiculo_abstracao.py)
